Screening Multiple Models for Predicting Volcanic Explosivity Index (VEI)

Yasin Khadem Charvadeh

In this analysis, I utilize a #TidyTuesday dataset on volcanic eruptions to evaluate multiple modeling approaches with the objective of identifying the algorithm with optimal predictive performance. Model comparison is a valuable methodology applied primarily when prior knowledge about which approach might yield superior accuracy on unseen data is limited. Here, I implement various preprocessing tools, including natural language processing methods and Adaptive Synthetic (ADASYN) sampling to prepare the dataset, and experiment with diverse algorithms such as support vector machines, naive Bayes classifiers, ensemble tree methods, and neural networks.

Pre-processing and Modelling

  • Data splitting
set.seed(101)

split_data <- initial_split(full_data, strata = vei_cat)

data_train <- training(split_data)

data_test <- testing(split_data)

set.seed(201)

vol_folds <- vfold_cv(data_train, strata = vei_cat)
  • Data preprocessing and feature engineering
models_rec <- recipe(vei_cat ~ ., data = data_train) %>%
  update_role(volcano_number, new_role = 'id variable') %>%
  update_role(vei, new_role = 'numeric vei') %>%
  update_role(country, new_role = 'country name') %>%
  step_normalize(latitude, longitude, elevation) %>% 
  step_other(tectonic_settings, threshold = tune()) %>%
  step_dummy(tectonic_settings, one_hot = TRUE) %>%
  step_tokenize(rock_type) %>%
  step_tf(rock_type) %>%
  step_adasyn(vei_cat, over_ratio = tune()) %>% 
  step_tomek(vei_cat)
  • Model specification
dec_tree_spec <-
  decision_tree(cost_complexity = tune(),
                min_n = tune()) %>%
  set_engine("rpart") %>%
  set_mode("classification")

rf_spec <-
  rand_forest(mtry = tune(),
              min_n = tune(),
              trees = 2000) %>%
  set_engine("ranger") %>%
  set_mode("classification")

nb_spec <- 
  naive_Bayes(smoothness = tune(),
              Laplace = tune()) %>%
  set_engine("klaR") %>% 
  set_mode("classification")

xgb_spec <-
  boost_tree(
    tree_depth = tune(),
    trees = tune(),
    learn_rate = tune(),
    mtry = tune(),
    min_n = tune(),
    loss_reduction = tune(),
    sample_size = tune()
  ) %>%
  set_engine("xgboost") %>%
  set_mode("classification")
  
nnet_spec <-
  mlp(hidden_units = tune(),
      penalty = tune(),
      epochs = tune()) %>%
  set_engine("nnet") %>%
  set_mode("classification")

svm_spec <-
  svm_rbf(cost = tune(), rbf_sigma = tune()) %>%
  set_engine("kernlab") %>%
  set_mode("classification")

  • Creating a workflow set

  • Finalizing hyperparameters of some models (if necessary)

race_ctrl <-
   control_race(
     save_workflow = TRUE,
     save_pred = TRUE,

race_results <-
   models_wf %>%
   workflow_map(
      "tune_race_anova",
      seed = 250,
      resamples = vol_folds,
      grid = 20,
      metrics = cls_metrics,
      control = race_ctrl
   )

The model's performance is clearly unsatisfactory, which was anticipated due to two main factors: (1) the dataset is unbalanced, and even applying the ADASYN algorithm yielded only limited improvement; (2) the dataset contains relatively few predictors. Collecting additional data and expanding the set of predictors would likely enhance the model's performance.
